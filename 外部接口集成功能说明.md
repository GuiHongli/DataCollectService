# 外部接口集成功能说明

## 功能概述

在DataCollectService服务中集成了外部接口调用功能，用于在采集任务下发时检查应用是否为新应用，并根据检查结果自动添加自定义参数。

## 实现的功能

### 1. 外部接口调用服务

- **服务类**: `ExternalApiService` 和 `ExternalApiServiceImpl`
- **控制器**: `ExternalApiController`
- **DTO类**: `AppCheckRequest` 和 `AppCheckResponse`

### 2. 配置

在 `application.yml` 中添加了外部接口配置：

```yaml
# 外部接口配置
external:
  api:
    host: http://localhost:9000
```

### 3. 集成到采集任务下发流程

在 `CollectTaskProcessServiceImpl.getTaskCustomParams()` 方法中集成了 `checkAppIsNew` 方法：

- 根据用例的APP字段获取应用名称
- 通过用例编号是否包含 `_ios_` 来判断是否为iOS应用
- 为每个用例单独调用外部接口，支持同一个APP的iOS和非iOS版本
- 根据返回的 `is_new` 值添加自定义参数：
  - 如果 `is_new` 为 `true`，添加参数 `key: "isnew", value: "1"`
  - 如果 `is_new` 为 `false`，添加参数 `key: "isnew", value: "0"`

## 接口规范

### 请求格式

**URL**: `{host}/api/apps/check_is_new`

**方法**: POST

**请求体**:
```json
[
  {
    "app_name": "微信",
    "is_ios": false
  }
]
```

### 响应格式

```json
{
  "message": "success",
  "data": [
    {
      "app_name": "微信",
      "is_ios": false,
      "is_new": false
    }
  ]
}
```

## 使用方式

### 1. 直接调用外部接口

```bash
curl -X POST http://localhost:8080/api/external/check-app-is-new \
  -H "Content-Type: application/json" \
  -d '[{"app_name": "微信", "is_ios": false}]'
```

### 2. 在采集任务中自动调用

当创建采集任务时，系统会自动：

1. 获取任务中所有用例的APP字段
2. 根据用例编号判断是否为iOS应用（包含 `_ios_` 关键字）
3. 为每个用例单独调用外部接口检查应用是否为新应用（支持同一APP的iOS和非iOS版本）
4. 将检查结果作为自定义参数添加到任务中

## 日志记录

系统会记录以下关键日志：

- 外部接口调用请求和响应
- 用例的APP信息和is_new状态
- 自定义参数的添加过程

## 错误处理

- 外部接口调用失败时，不会影响采集任务的正常创建
- 会在日志中记录错误信息
- 如果无法获取is_new状态，则不会添加isnew参数

## 注意事项

1. 确保外部接口服务正常运行
2. 外部接口的host配置需要正确
3. 用例编号中包含 `_ios_` 关键字才会被识别为iOS应用
4. 只有APP字段不为空的用例才会参与检查
5. 同一个APP可以同时存在iOS和非iOS版本，系统会为每个用例单独检查
