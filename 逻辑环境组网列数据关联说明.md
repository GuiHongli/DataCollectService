# 逻辑环境组网列数据关联说明

## 概述

已确认逻辑环境管理页面中"逻辑组网"列的数据直接关联到网络类型管理的后台数据表，实现了数据的完全统一和实时同步。

## 数据关联实现

### 1. 后台数据关联

#### LogicEnvironmentController 更新
- **数据源**: 使用 `NetworkTypeService` 和 `NetworkType` 实体
- **关联查询**: 通过 `logic_environment_network` 关联表查询网络类型数据
- **数据获取**: `getNetworks` 方法直接查询 `network_type` 表

```java
// 获取组网详细信息（使用网络类型数据表）
QueryWrapper<NetworkType> networkQuery = new QueryWrapper<>();
networkQuery.in("id", networkIds);
List<NetworkType> networks = networkTypeService.list(networkQuery);
```

#### 数据表关联关系
- **主表**: `network_type` (网络类型数据表)
- **关联表**: `logic_environment_network` (逻辑环境与网络关联表)
- **查询逻辑**: 先查询关联表获取网络ID，再查询网络类型表获取详细信息

### 2. 前端数据显示

#### 逻辑组网列实现
```vue
<el-table-column label="逻辑组网" min-width="200">
  <template #default="scope">
    <div v-if="scope.row.networkList && scope.row.networkList.length > 0">
      <div v-for="network in scope.row.networkList" :key="network.id" style="margin-bottom: 8px;">
        <el-tag size="small" type="success">
          {{ network.name }}
        </el-tag>
      </div>
    </div>
    <span v-else style="color: #909399;">暂无组网</span>
  </template>
</el-table-column>
```

#### 数据来源
- **接口**: `/logic-environment/{id}/network`
- **数据源**: 直接来自 `network_type` 表
- **显示内容**: 网络名称、描述等信息

### 3. 数据同步机制

#### 实时同步
- 网络类型数据的任何修改都会立即反映到逻辑环境组网列
- 禁用状态的网络类型不会出现在逻辑环境组网列中
- 新增网络类型可以立即关联到逻辑环境

#### 状态管理
- 通过 `network_type.status` 字段控制网络类型的可见性
- 只有启用状态 (`status = 1`) 的网络类型才会显示在逻辑环境组网列中

## 数据流程

### 1. 数据查询流程
```
逻辑环境列表页面
    ↓
调用 /logic-environment/page 接口
    ↓
获取逻辑环境基本信息
    ↓
调用 /logic-environment/{id}/network 接口
    ↓
查询 logic_environment_network 关联表
    ↓
根据网络ID查询 network_type 表
    ↓
返回网络类型详细信息
    ↓
在逻辑组网列中显示
```

### 2. 数据修改流程
```
网络类型管理页面修改数据
    ↓
更新 network_type 表
    ↓
逻辑环境组网列自动同步更新
    ↓
用户看到最新的网络信息
```

## 技术实现细节

### 1. 后台实现

#### 服务层
- **NetworkTypeService**: 提供网络类型数据操作
- **LogicEnvironmentNetworkService**: 管理逻辑环境与网络的关联关系

#### 控制器层
- **LogicEnvironmentController.getNetworks()**: 获取逻辑环境关联的网络信息
- **数据查询**: 先查关联表，再查网络类型表
- **数据转换**: 将网络类型数据转换为前端需要的格式

### 2. 前端实现

#### 数据获取
- **loadData()**: 加载逻辑环境列表数据
- **networkList**: 每个逻辑环境对象包含关联的网络列表
- **显示逻辑**: 遍历 networkList 显示网络标签

#### 数据展示
- **标签显示**: 使用 `el-tag` 组件显示网络名称
- **空状态**: 当没有关联网络时显示"暂无组网"
- **样式**: 使用绿色标签表示网络信息

### 3. 数据库设计

#### 表结构
```sql
-- 网络类型表
CREATE TABLE network_type (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    status INT DEFAULT 1,
    create_time DATETIME,
    update_time DATETIME,
    deleted INT DEFAULT 0
);

-- 逻辑环境网络关联表
CREATE TABLE logic_environment_network (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    logic_environment_id BIGINT NOT NULL,
    logic_network_id BIGINT NOT NULL,
    create_time DATETIME,
    update_time DATETIME,
    deleted INT DEFAULT 0
);
```

#### 关联关系
- `logic_environment_network.logic_network_id` → `network_type.id`
- 多对多关系：一个逻辑环境可以关联多个网络类型
- 一个网络类型可以被多个逻辑环境使用
