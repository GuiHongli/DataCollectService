# 应用检查集成功能说明

## 功能概述

在DataCollectService服务中新增了应用检查集成功能，在新建采集任务下发前会自动调用外部接口检查应用是否为新应用，并根据检查结果在用例下发时添加相应的自定义参数。

## 新增文件

### 1. DTO类
- `AppCheckRequest.java` - 应用检查请求DTO
- `AppCheckResponse.java` - 应用检查响应DTO

### 2. 服务类
- `ExternalApiService.java` - 外部接口调用服务接口
- `ExternalApiServiceImpl.java` - 外部接口调用服务实现类

### 3. 控制器
- `ExternalApiController.java` - 外部接口调用控制器

## 配置说明

### application.yml配置
```yaml
# 外部接口配置
external:
  api:
    host: http://localhost:9000
```

## 功能实现

### 1. 外部接口调用
- **接口地址**: `{host}/api/apps/check_is_new`
- **请求方法**: POST
- **请求体格式**:
```json
[
  {
    "app_name": "微信",
    "is_ios": false
  }
]
```
- **响应体格式**:
```json
{
  "message": "success",
  "data": [
    {
      "app_name": "微信",
      "is_ios": false,
      "is_new": false
    }
  ]
}
```

### 2. 集成逻辑
在`CollectTaskProcessServiceImpl.buildTestCaseList()`方法中：

1. **收集应用信息**: 遍历所有用例，提取APP字段和用例名称
2. **判断iOS平台**: 通过用例名称是否包含`_ios_`来判断是否为iOS应用
3. **调用外部接口**: 批量调用外部接口检查应用是否为新应用
4. **添加自定义参数**: 根据检查结果在用例信息中添加`isnew`参数
   - `is_new = true` → `{"isnew": "1"}`
   - `is_new = false` → `{"isnew": "0"}`

### 3. 自定义参数格式
在`TestCaseExecutionRequest.TestCaseInfo`中新增`customParams`字段，用于存储用例级别的自定义参数。

## API接口

### 检查应用是否为新应用
- **URL**: `/api/external/check-app-is-new`
- **方法**: POST
- **请求体**: `List<AppCheckRequest>`
- **响应**: 标准响应格式

## 注意事项

1. **外部接口依赖**: 需要确保外部接口服务正常运行
2. **异常处理**: 如果外部接口调用失败，会记录错误日志但不影响任务下发
3. **性能考虑**: 批量调用外部接口，避免频繁请求
4. **日志记录**: 详细记录应用检查过程和结果

## 使用示例

当创建采集任务时，系统会自动：
1. 检查用例的APP字段（如"微信"）
2. 根据用例名称判断平台（包含"_ios_"则为iOS）
3. 调用外部接口检查应用是否为新应用
4. 在用例下发时添加相应的`isnew`参数

例如：
- 用例名称: "微信登录测试_ios_001"
- APP字段: "微信"
- 检查结果: is_new = true
- 添加参数: `{"isnew": "1"}`
