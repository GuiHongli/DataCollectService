# 逻辑组网与网络类型数据统一确认

## 概述

已确认逻辑环境管理页面的逻辑组网数据与网络类型管理页面的后台数据使用同一份数据源，实现了数据的完全统一。

## 数据统一实现

### 1. 后台控制器统一

#### LogicNetworkController 更新
- **数据源**: 使用 `NetworkTypeService` 和 `NetworkType` 实体
- **接口路径**: 保持 `/logic-network/*` 路径不变，确保前端兼容性
- **数据过滤**: `/list` 接口只返回启用状态的网络类型 (`status = 1`)

#### NetworkTypeController
- **数据源**: 使用 `NetworkTypeService` 和 `NetworkType` 实体
- **接口路径**: `/network-type/*`
- **完整功能**: 支持所有网络类型的CRUD操作

### 2. 前端数据获取统一

#### 逻辑环境管理页面
```javascript
// 获取网络选项数据
const loadNetworkOptions = async () => {
  const res = await request({
    url: '/network-type/list',  // 使用网络类型接口
    method: 'get',
  })
  networkOptions.value = res.data
}

// 获取可用网络数据
const loadAvailableNetworks = async () => {
  const res = await request({
    url: '/network-type/list',  // 使用网络类型接口
    method: 'get',
  })
  availableNetworks.value = res.data
}
```

#### 网络类型管理页面
```javascript
// 获取网络类型数据
const loadData = async () => {
  const res = await request({
    url: '/network-type/page',  // 使用网络类型接口
    method: 'get',
    params: { current: pagination.current, size: pagination.size }
  })
  tableData.value = res.data.records
}
```

### 3. 数据表统一

#### 统一数据表
- **主表**: `network_type` 表
- **字段**: `id`, `name`, `description`, `status`, `create_time`, `update_time`, `deleted`
- **状态管理**: 通过 `status` 字段控制启用/禁用

#### 关联关系
- 逻辑环境与网络的关联通过 `logic_environment_network` 表维护
- 关联表引用 `network_type` 表的ID

## 接口映射关系

| 功能 | 逻辑组网接口 | 网络类型接口 | 数据源 |
|------|-------------|-------------|--------|
| 查询列表 | `/logic-network/list` | `/network-type/list` | `network_type` 表 |
| 分页查询 | `/logic-network/page` | `/network-type/page` | `network_type` 表 |
| 创建 | `/logic-network` | `/network-type` | `network_type` 表 |
| 更新 | `/logic-network/{id}` | `/network-type/{id}` | `network_type` 表 |
| 删除 | `/logic-network/{id}` | `/network-type/{id}` | `network_type` 表 |

## 数据一致性保证

### 1. 统一数据源
- 所有网络相关数据都存储在 `network_type` 表中
- 逻辑组网接口和网络类型接口操作同一张表
- 确保数据的唯一性和一致性

### 2. 状态过滤
- 逻辑组网接口只返回启用状态的网络类型 (`status = 1`)
- 网络类型管理接口可以查看所有状态的数据
- 禁用状态的网络类型不会出现在逻辑组网选择列表中

### 3. 实时同步
- 任何对网络类型的修改都会立即反映到所有相关接口
- 逻辑环境管理页面和网络类型管理页面显示相同的数据
- 数据修改无需手动刷新，自动同步
