# 用例集字段扩展说明

## 修改日期
2025-10-11

## 修改背景
根据业务需求，需要在用例集详情页面显示更完整的用例信息，包括 APPEN、模型场景、手机OS类等字段。

## Excel 文件格式说明

上传的压缩包中的 `case.xlsx` 文件应包含以下 10 列（从第1列到第10列）：

| 列序号 | 列名 | 字段名 | 说明 | 是否必填 |
|-------|------|--------|------|---------|
| 1 | 用例名称 | name | 测试用例的名称 | 是 |
| 2 | 用例编号 | number | 测试用例的唯一编号 | 是 |
| 3 | 用例逻辑组网 | logicNetwork | 逻辑组网环境，多个用分号分隔 | 否 |
| 4 | 用例业务大类 | businessCategory | 业务分类 | 否 |
| 5 | 用例APP | app | 应用名称 | 否 |
| 6 | 用例APPEN | appEn | 应用英文名称 | 否 |
| 7 | 用例模型场景 | modelScenario | 模型场景描述 | 否 |
| 8 | 用例手机OS类 | phoneOsType | 手机操作系统类型 | 否 |
| 9 | 用例操作步骤 | testSteps | 详细的测试步骤 | 否 |
| 10 | 用例预期结果 | expectedResult | 预期的测试结果 | 否 |

## 修改内容

### 1. 数据库表结构修改

#### 新增字段
在 `test_case` 表中添加了以下字段：
- `app_en` varchar(100) - 用例APPEN
- `model_scenario` varchar(100) - 用例模型场景
- `phone_os_type` varchar(50) - 用例手机OS类

#### 相关 SQL 文件
- **迁移脚本**: `src/main/resources/db/add_test_case_new_fields.sql`
  - 用于现有数据库添加新字段
  
- **建表脚本**: `src/main/resources/db/add_test_case.sql`
  - 已更新，新建表时包含所有字段

### 2. 后端代码修改

#### TestCase 实体类
文件：`src/main/java/com/datacollect/entity/TestCase.java`

新增字段：
```java
@TableField("app_en")
private String appEn;

@TableField("model_scenario")
private String modelScenario;

@TableField("phone_os_type")
private String phoneOsType;
```

#### ExcelParser 解析工具类
文件：`src/main/java/com/datacollect/util/ExcelParser.java`

修改内容：
1. 更新列索引常量，添加新列定义
2. 修改 `buildTestCaseFromRow` 方法，解析新的三列数据
3. 更新空行检查列数从 7 改为 10

### 3. 前端页面修改

#### 用例集详情页面
文件：`DataCollectFront/src/views/test-case-set/detail.vue`

修改内容：
1. 在表格中新增三列显示：
   - APPEN（appEn）
   - 模型场景（modelScenario）
   - 手机OS类（phoneOsType）
2. 调整列宽和布局，提升显示效果
3. 用例名称列设置为固定列，横向滚动时保持可见

## 数据库升级步骤

### 方式一：执行迁移脚本（推荐）
```sql
-- 在现有数据库上执行
source src/main/resources/db/add_test_case_new_fields.sql
```

### 方式二：手动执行 SQL
```sql
ALTER TABLE `test_case`
ADD COLUMN `app_en` varchar(100) DEFAULT NULL COMMENT '用例APPEN' AFTER `app`,
ADD COLUMN `model_scenario` varchar(100) DEFAULT NULL COMMENT '用例模型场景' AFTER `app_en`,
ADD COLUMN `phone_os_type` varchar(50) DEFAULT NULL COMMENT '用例手机OS类' AFTER `model_scenario`;
```

## 兼容性说明

1. **向后兼容**: 新增字段均为可选字段（允许 NULL），不影响现有数据
2. **Excel 兼容**: 
   - 新上传的 Excel 文件应包含所有 10 列
   - 如果某列为空，系统会显示"未配置"提示
   - 旧格式的 Excel 文件可能导致解析错误，建议更新为新格式
