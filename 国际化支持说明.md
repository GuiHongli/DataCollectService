### 2. 前端页面文件

**文件**: `DataCollectFront/src/views/test-case-set/detail.vue`

#### 主要修改点

##### 2.1 表格列标题国际化
所有表格列的标题都使用了 `$t()` 函数：

```vue
<el-table-column prop="appEn" :label="$t('testCaseSet.appEn')" width="120">
  <template #default="scope">
    <span v-if="scope.row.appEn">{{ scope.row.appEn }}</span>
    <span v-else style="color: #909399;">{{ $t('collectTask.notConfigured') }}</span>
  </template>
</el-table-column>
```

##### 2.2 消息提示国际化
所有用户消息（ElMessage）都使用国际化：

```javascript
// 成功消息
ElMessage.success(t('testCaseSet.urlCopied'))

// 错误消息
ElMessage.error(t('testCaseSet.loadDataFailed'))
ElMessage.error(t('testCaseSet.testCaseSetIdRequired'))
```

##### 2.3 日期时间格式化支持多语言
根据当前语言环境格式化日期：

```javascript
const formatDateTime = (dateTime) => {
  if (!dateTime) return ''
  const currentLocale = locale.value === 'zh' ? 'zh-CN' : 'en-US'
  return new Date(dateTime).toLocaleString(currentLocale)
}
```

## 语言切换功能

系统支持动态语言切换，用户可以通过页面右上角的语言切换按钮在中英文之间切换。

### 切换效果

1. **表格列标题**：自动切换为对应语言
2. **空值提示**："未配置" → "Not configured"
3. **消息提示**：所有 ElMessage 消息都切换为对应语言
4. **日期时间**：自动使用对应地区的日期格式
   - 中文：`2025/10/14 14:30:25`
   - 英文：`10/14/2025, 2:30:25 PM`

### 语言持久化

用户选择的语言会保存到浏览器的 localStorage 中，下次访问时自动应用上次选择的语言。

## 使用说明

### 添加新的国际化文本

如果需要添加新的国际化文本，按以下步骤操作：

#### 1. 在语言包中添加翻译
```javascript
// zh.js
testCaseSet: {
  newField: '新字段名称',
}

// en.js
testCaseSet: {
  newField: 'New Field Name',
}
```

#### 2. 在组件中使用
```vue
<!-- 在模板中 -->
<el-table-column :label="$t('testCaseSet.newField')" />

<!-- 在脚本中 -->
<script>
export default {
  setup() {
    const { t } = useI18n()
    
    // 在消息中使用
    ElMessage.success(t('testCaseSet.newField'))
  }
}
</script>
```

## 注意事项

1. **翻译一致性**：确保同一概念在不同位置使用相同的翻译 key
2. **默认语言**：系统默认语言为中文（zh）
3. **后备语言**：如果翻译缺失，会自动使用中文作为后备
4. **控制台日志**：开发环境下，console.error 中的日志保持英文，便于问题追踪

## 扩展支持

如需添加其他语言（如日语、韩语等），按以下步骤：

1. 在 `src/locales/` 目录下创建新的语言文件（如 `ja.js`、`ko.js`）
2. 复制 `zh.js` 的结构，翻译所有文本
3. 在 `src/i18n/index.js` 中注册新语言
4. 在语言切换组件中添加新语言选项
