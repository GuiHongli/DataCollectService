# 用户登录系统说明

## 功能概述

系统已实现完整的用户登录和权限管理功能，包括：

1. **用户登录**：支持用户名和密码登录，使用JWT Token进行认证
2. **密码加密**：使用BCrypt算法对密码进行加密存储
3. **用户管理**：只有admin角色可以新增、删除用户
4. **路由守卫**：前端自动拦截未登录请求，跳转到登录页
5. **权限控制**：后端API接口通过拦截器验证Token

## 数据库初始化

### 用户表结构

用户表已添加到 `init.sql` 中，包含以下字段：
- `id`: 主键ID
- `username`: 用户名（唯一）
- `password`: 密码（BCrypt加密）
- `role`: 角色（admin/user）
- `status`: 状态（0-禁用，1-启用）
- `create_by`: 创建人
- `update_by`: 修改人
- `create_time`: 创建时间
- `update_time`: 修改时间
- `deleted`: 逻辑删除标记

### 初始化默认用户

系统在 `init.sql` 中提供了默认用户数据，但密码需要重新生成。

**重要**：SQL中的BCrypt密码是示例，实际部署时需要重新生成。

#### 方法1：使用Java代码生成密码

可以创建一个临时工具类生成BCrypt密码：

```java
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;

public class PasswordGenerator {
    public static void main(String[] args) {
        BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
        String password = "admin123"; // 要加密的密码
        String encoded = encoder.encode(password);
        System.out.println("加密后的密码: " + encoded);
    }
}
```

#### 方法2：使用在线工具

访问 https://bcrypt-generator.com/ 生成BCrypt密码

#### 方法3：通过API创建

系统启动后，可以通过API创建admin用户（需要先手动在数据库中插入一个admin用户，或者临时禁用权限检查）

## 默认用户

系统提供了两个默认用户（密码需要重新生成）：

1. **admin** - 管理员
   - 用户名：admin
   - 密码：admin123（需要重新生成BCrypt密码）
   - 角色：admin

2. **user** - 普通用户
   - 用户名：user
   - 密码：admin123（需要重新生成BCrypt密码）
   - 角色：user

## API接口

### 认证接口

#### 1. 用户登录
- **URL**: `/api/auth/login`
- **方法**: POST
- **请求体**:
```json
{
  "username": "admin",
  "password": "admin123"
}
```
- **响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "username": "admin",
    "role": "admin"
  }
}
```

#### 2. 获取当前用户信息
- **URL**: `/api/auth/info`
- **方法**: GET
- **请求头**: `Authorization: Bearer {token}`

#### 3. 退出登录
- **URL**: `/api/auth/logout`
- **方法**: POST

### 用户管理接口（需要admin权限）

#### 1. 创建用户
- **URL**: `/api/user`
- **方法**: POST
- **权限**: 仅admin
- **请求体**:
```json
{
  "username": "newuser",
  "password": "password123",
  "role": "user"
}
```

#### 2. 分页查询用户
- **URL**: `/api/user/page`
- **方法**: GET
- **参数**: current, size, username, role

#### 3. 更新用户
- **URL**: `/api/user/{id}`
- **方法**: PUT
- **权限**: admin可以修改所有用户，普通用户只能修改自己

#### 4. 删除用户
- **URL**: `/api/user/{id}`
- **方法**: DELETE
- **权限**: 仅admin

#### 5. 修改密码
- **URL**: `/api/user/{id}/password`
- **方法**: POST
- **权限**: admin可以修改所有用户密码，普通用户只能修改自己的密码

## 前端页面

### 登录页面
- **路径**: `/login`
- **功能**: 用户登录，输入用户名和密码

### 用户管理页面
- **路径**: `/user/index`
- **功能**: 
  - 查看用户列表
  - 新增用户（仅admin可见）
  - 编辑用户信息
  - 删除用户（仅admin可见）
  - 修改密码

## 权限控制

### 后端权限控制

1. **认证拦截器** (`AuthInterceptor`): 拦截所有API请求，验证JWT Token
2. **用户管理接口**: 只有admin角色可以创建和删除用户
3. **用户信息修改**: admin可以修改所有用户，普通用户只能修改自己

### 前端权限控制

1. **路由守卫**: 自动检查登录状态，未登录跳转到登录页
2. **按钮权限**: 根据用户角色显示/隐藏功能按钮
3. **Token管理**: 自动在请求头中添加Token，Token过期自动跳转登录

## 安全特性

1. **密码加密**: 使用BCrypt算法，每次加密结果不同，但可以验证
2. **JWT Token**: 24小时有效期，存储在localStorage
3. **自动登出**: Token过期或无效时自动清除登录状态
4. **权限验证**: 后端接口验证用户角色，防止越权操作

## 使用说明

### 首次部署

1. 执行数据库初始化脚本 `init.sql`
2. 使用工具生成BCrypt密码，更新SQL中的密码字段
3. 或者系统启动后，通过API创建admin用户（需要先手动插入一个admin用户）

### 日常使用

1. 访问系统首页，自动跳转到登录页
2. 使用admin账号登录
3. 在用户管理页面创建新用户
4. 新用户可以使用自己的账号登录

## 注意事项

1. **密码安全**: 生产环境请修改默认密码
2. **Token安全**: JWT密钥存储在代码中，生产环境建议配置化
3. **HTTPS**: 生产环境建议使用HTTPS传输
4. **密码策略**: 建议添加密码复杂度要求（当前仅要求6位以上）
